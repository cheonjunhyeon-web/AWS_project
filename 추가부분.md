## step1
2026 클라우드 해킹 트렌드 <br>
1. SQL 인젝션 방식 대상 변경
  - LLM 인젝션 : 웹 서비스 뒤에 붙은 AI 모델에 악의적인 프롬프트를 넣어 내부 DB 정보를 빼내거나, 시스템 명령을 실행하는 방식
  - 자동화된 스캔 : AI 해킹 툴이 24시간 내내 전 세계 웹사이트의 SQL 취약점을 자동으로 찾기 때문에 방심하면 안됨.
2. 서버리스 및 클라우드 네이티브 공격
  - Event Injection: S3나 SNS에서 발생하는 이벤트를 가로채거나 조작하여 람다 함수가 엉뚱한 동작을 하게 만든다.
  - 권한 탈취(IAM Abuse) : 람다에 부여된 과도한 권한(FullAccess등)을 이용해 클라우드 인프라 전체를 장악하는 방식
3. API 기반 DDoS 및 비즈니스 로직 공격
  단순히 트래픽만 쏘는 게 아닌, 정상적인 API 호출처럼 위장하여 시스템을 마비
  - Slowloris : 아주 느리게 요청을 보내서 서버의 연결 자원을 다 써버리는 공격
  - 인벤토리 고갈 : 커머스 사이트에서 결제는 안 하고 장바구니에만 수만 개를 담아 재고를 0으로 만들어버리는 로직 공격

## step 2 
방어 방법 
1. 입력값 검증 (Input Validation) : 모든 람다 함수 시작 부분에 데이터의 형식이 맞는지 확인하는 코드 입력 <br>
2. 스키마 체크 ( Schema Enforcement): Json 데이터의 키와 값의 타입(숫자인지, 문자인지, 길이는 적당한지)를 엄격하게 제한한다. <br>
3. 최소 권한 원칙(Least Privilege): 람다가 DB를 삭제할 권한이 없다면, 공격 코드가 주입되어도 실행되지 않습니다. <br>
